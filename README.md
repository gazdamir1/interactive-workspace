Интерактивная рабочая область

Проект собран на React, TypeScript, Vite с дополнением SCSS. Тесты на Vitest.

Архитектура
src/
├── components/
│ ├── Workspace/ # рабочая область (контейнер для объектов)
│ ├── ObjectItem/ # отдельный объект (с логикой перетаскивания)
│ ├── InfoPanel/ # боковая панель с информацией об активном объекте
│ └── UI/ # переиспользуемые компоненты (Button)
├── store/
├── types/ - глобальные типы и интерфейсы
├── data/ - моковые данные
├── utils/ - функции для работа с localStorage
├── context/ - React-контекст (для передачи ref рабочей области)
├── test/ - предустановки тестовой среды
└── App.tsx

Модули

- Store (Zustand) хранит массив объектов, выделенный ID, методы для изменения статуса, позиции, сброса и рандомизации.
- Workspace отображает все объекты, передаёт им ref через контекст для корректного расчёта координат при перетаскивании.
- ObjectItem отвечает за рендер одного объекта, обработку кликов и touch-событий, перетаскивание (с учётом disabled-статуса), анимации.
- InfoPanel появляется при выделении объекта, позволяет заблокировать/разблокировать его (переключение между inactive и disabled).

Принятые решения

- При "пробелах" в ТЗ, подстраивал логику по своему пониманию.
- Дизайн по своему усмотрению, чуть усложнил, чтобы визуальное отображение выглядило понятнее.
- Добавил кнопку отката до изначальной версии доски, так как при возможности блокировки, вернуть кнопку в изначальное состояние именно на доске - невозможно.
- Для адаптивности добавил отступ слева, так как при сужении окна по вертикали появляется скролл, который изначально "дёргает" рабочую область и делаёт её ассимитричной.
- Сужение заглавного текста и блокировка переходов на кнопках, также для избегания эффекта "дёрганья" при сужении окна.
- Выполение всех дополнительных заданий.

Возможные улучшения.

- Демо. У меня не вышло, так как прошлые решения такие как Github Pages, Nutlify - либо не загружаются, либо перешли на платную основу.
- Плавный переход при смещении объектов.
- Измение дизайна на "тёмный".
